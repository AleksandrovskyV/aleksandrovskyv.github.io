Суть html документа: интерфейс из кнопок разной ширины во Flexbox-контейнере (#buttonContainer).

Основные заметки:
1. Геометрия: Родитель .fswrap центрирует всё через align-items: center.
2. Строгое поведение на мобилках через MOBILE_TARGET_ROW = 8.
3. Подсчёт строк через getRowData() используя new Set(offsetTop).
4. Принципы удаления: removeOverflowBtnsMobile использует Backtracking (откат). Если удаление кнопки схлопывает 8 строк в 7 — возвращаем кнопку на место и выходим. 
5. Удаляем только кнопки с data-st="lock" - те что заблокированы от пользователя
6. Работа с конечной строкой через checkLastRow - разное поведение applyFix для мобилки и десктопа
7. Ghosts: Вместо перемещения кнопки-соседа, мы клонируем её, делаем клон прозрачным (opacity: 0), а оригинал сажаем в endGroup. Это держит высоту и не дает верстке прыгать.
8. Если applyFix (подселение или объединение ряда) меняет количество строк (стало 7 или 9) — делаем Rollback .

Задачи/проблемы:
Текущая логика работает без косяков на первой инициализации.
Она корректно расставляет кнопки в мобильной и десктоп версии.
Но есть пара архитектурный проблем / косячков, которые нужно исправить.

Сейчас проблема в попытке на desktop повторно вызвать rebuildGUI (840-855 строчка) нажатием клавиши "M".
Этот метод в теории должен вернуть кнопки из переменной SOURCE_BUTTONS, затем перемешать, и выполнить свою исходную логику. 

Основной принцип выравнивания описан в блоке GUI Preload (341-573 строчки). 
Вся инициализация в GUI Updater (770-855 строчки).

Стоит обратить внимание на ghost класс кнопок, которые создаются для сохранения верстки. 
В теории для них прописана логика защиты в restoreSourceBtns (809-827), но увы только для группы endGroup (пара кнопок в конце)
 - для того, чтобы ghost не появлялся при resize (повороте) на телефоне. 

Решение на поверхности - добавить переменную CURRENT_BUTTONS для сохранения текущих кнопок после перетасовок, а SOURCE_BUTTONS для полного restore исходника (допустим для M сброса). Альтернатива - сохранить на инициализации кнопки в переменную с именем SOURCE_BUTTONS с атрибутом source, а после перемешивания пометить часть из них как current). И после в initPage / hardShuffle - использовать все кнопки массива, а при обновление (resize) только те, что были помечены current. В таком случае у нас не будет раздуваться код на разные массивы для разных задач, что меня больше устраивает.
